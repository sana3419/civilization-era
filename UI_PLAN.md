# 交互重构规划：铁锈战争式选中上下文系统

> 试玩反馈驱动（2026-06-12）：建造按钮应在选中工人后才显示；点击单位/建筑应查看状态；
> 整体模仿铁锈战争（Rusted Warfare，下称 RW）的单位功能逻辑。
> 状态：**规划稿，待确认后实施**。

## 一、RW 交互模型调研结论

来源：Steam 社区讨论、PCGamingWiki、RW 更新日志（见文末）。核心机制：

1. **一切功能挂在"选中"上**：屏幕上没有常驻的全局功能按钮；
   选中什么，底部/侧栏就出现什么功能。
2. **选中列表按类型分组**（侧栏 chips）：多选时显示"类型×数量"图标列表，
   左键点 chip = 只保留该类型，右键点 chip = 从选中中剔除该类型。
3. **建造单位（builder）选中 → 出现建造菜单**：建造按钮属于工程单位的能力，
   不是全局 UI；点按钮进入放置模式。
4. **工厂（建筑）可选中 → 出现生产按钮**：训练/生产从所选建筑触发，
   支持队列（Shift/Ctrl 批量 +5/+10），右键地面设集结点。
5. **选中单位显示状态**：HP（护盾）等信息随选中显示。
6. **右键 = 上下文命令**：移动/攻敌/（本作已有：登墙/修理/采集）。

## 二、映射到本作的设计

### 2.1 选中模型扩展

- 现状：只能选单位。
- 新增：**建筑可选中**（左键点击建筑）。选中互斥：要么一组单位，要么一座建筑。
- 优先级：点击处有己方单位 → 选单位；否则有建筑 → 选建筑；否则空选（城门开关
  移入建筑面板按钮，取消"空点开门"的隐藏操作）。

### 2.2 底部上下文面板（替换常驻建造栏）

| 选中内容 | 面板显示 |
|----------|----------|
| 无 | 空（仅保留资源栏/小地图/状态行） |
| ≥1 工人 | **建造菜单**（分类两行：经济 伐木场/采石场/农田/房屋/仓库 · 军防 兵营/射箭场/马厩/箭塔/栅栏/木门/石墙/石门/攻城工坊）+ 命令按钮 [停止] [修理模式] |
| 军事单位 | 命令按钮 [攻击移动] [停止] [登墙] + **阵型一排**（8 钮，替代纯 F1-F8）+ 类型分组 chips |
| 混合选中 | 军事面板 + 建造菜单折叠入 [建造▼]（工人在选中内时） |
| 兵营/射箭场/马厩/攻城工坊 | **训练按钮迁移至此**（出兵位置=该建筑旁，Shift=+5 队列后续版本）+ 集结点提示（选中建筑后右键地面=设集结点）+ HP |
| 城门/石门 | [开/关] 按钮 + HP |
| 其他建筑 | 名称 + HP（+ 农田：产出进度） |
| 单选任意单位 | 状态卡：HP/士气/状态/携带/类型（现状态行升级） |

### 2.3 类型分组 chips（RW 侧栏行为）

多选单位时面板左侧显示 `工人×6` `民兵×4` `弓手×2` 图标钮：
左键 chip = 只选该类型；右键 chip = 剔除该类型。编队（Ctrl+1~9）不变。

### 2.4 命令语义（对照 DESIGN.md 快捷键表，本期实现范围）

- [移动]：即右键（不做按钮）
- [攻击移动] A：= command_move（军队行进自动索敌，语义已等价）
- [停止] S：**新增 command_stop**（清目标/状态→IDLE）
- [登墙] / [修理]：已有右键智能分派，面板加显式按钮（点按钮后下一次左键点目标）
- 守卫 H / 巡逻 P：推迟
- 生产队列（Shift 批量）、集结点多路点：推迟（RW 也不支持多路点集结）

### 2.5 已随本规划落地的修复（同一方向，已提交）

- 平原食物**不可徒手采集**（右键平原=移动；修复"单位到达后又返回"的采集往返误触）
- **农田定时产粮**（5 食物/10 秒，正式工人分配系统前的切片实现）
